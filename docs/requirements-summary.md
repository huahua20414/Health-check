# 需求摘要

来源：`/Users/huahua/Downloads/副本需求陈述书-东软熙心健康体检管理系统.docx`

文档明确的基础要求：

- 项目实现用户端体检预约全流程。
- 医生端根据数据生成体检报告全流程。
- 用户可以查看自己的体检报告。
- 后端使用 Go。
- 前端使用 Vue3、ElementPlus。
- 前后端分离接口设计。
- 数据库使用 MySQL。

当前实现按作业提交优先级采用 MVP：

- 用户、医生、管理员使用邮箱、密码和邮箱验证码登录。
- 用户可查看体检套餐、选择预约类型和体检机构并创建预约。
- 医生可查看预约并提交体检报告。
- 用户可查看自己的预约和报告。
- 使用 Docker Compose 启动 MySQL、后端、前端。
- 使用 Makefile 统一启动和插入模拟数据。

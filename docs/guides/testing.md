# 测试与交付检查

## 基础检查

前端改动至少执行：

```bash
cd frontend && npm run build
```

后端业务、接口、权限、数据模型改动至少执行相关包测试：

```bash
cd backend && go test ./internal/handlers
```

较大修改后执行：

```bash
cd backend && go test ./...
cd frontend && npm run build
```

如果本地环境限制导致完整测试失败，必须说明失败原因，并尽量运行可覆盖本次改动的定向测试。

## 部署与种子数据检查

涉及 Docker、端口、部署、种子数据时执行：

```bash
docker compose up -d --build
make seed
```

检查项：

- 前端端口为 `5174`。
- 后端对外端口为 `8081`。
- `make seed` 可重复执行。
- 模拟用户、医生、套餐、机构、号源、预约、候补、报告之间关系完整。
- 号源分布应覆盖未来两周，且存在不均匀分布和部分已满时间段。

## 权限回归

需要验证：

- 未登录访问受保护接口返回 `401`。
- 用户访问医生或管理员接口返回 `403`。
- 医生访问非本人预约或报告返回 `403`。
- 管理员可以审核医生。
- 前端用户、医生、管理员看到的菜单不同。

## 功能真实性回归

- 所有按钮必须有真实接口或真实前端行为，不允许只展示假数据。
- 长列表需要分页或明确分页策略。
- 修改和新增类操作优先用弹窗，且保存后刷新真实数据。
- 只读展示页面不能伪装成可操作功能。

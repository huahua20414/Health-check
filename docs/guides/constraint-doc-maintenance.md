# 约束文档维护

## 文档结构

本项目参考 `hsxa-go/backend` 的约束文档组织方式：

- `AGENTS.md`：只放最高优先级规则、加载入口、项目速记和必须遵守的硬规则。
- `CLAUDE.md`：只负责强制加载 `AGENTS.md`。
- `docs/guides/`：按主题拆分详细规则，避免单个入口文档过长。

## 维护规则

- 技术栈、端口、入口文件、目录职责发生变化时，必须同步更新 `AGENTS.md` 和相关 guide。
- 业务规则发生变化时，优先更新对应主题 guide，再改代码。
- 新增长期有效的硬规则时，先判断放在 `AGENTS.md` 还是某个 guide：
  - 所有任务都必须知道的规则放 `AGENTS.md`。
  - 只在特定主题触发的规则放对应 guide。
- 不要把一次性任务记录、临时排查过程、长日志放进约束文档。
- 不要在多个文档重复维护同一条细则；`AGENTS.md` 保留入口链接，细节放 guide。

## 新增 guide 的标准

新增 guide 时必须包含：

- 触发条件：什么时候必须读这个 guide。
- 规则正文：必须做什么、禁止做什么。
- 验证方式：如何证明遵守了规则。
- 相关文件：涉及的核心目录或入口。

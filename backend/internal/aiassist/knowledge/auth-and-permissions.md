# 认证与权限

## 登录注册

- 登录和注册使用邮箱验证码，不再采集或校验密码。
- 登录接口返回 JWT。
- 前端必须通过 `Authorization: Bearer <token>` 调用受保护接口。
- Redis 保存服务端 session，退出登录时删除 session。
- Redis 同时用于限流等服务端防刷能力。

## 角色权限

用户 `user`：

- 可以注册并登录。
- 可以查看体检套餐。
- 可以创建自己的预约。
- 只能查看自己的预约和报告。
- 不能访问医生端和管理端页面。
- 不能调用医生或管理员接口。

医生 `doctor`：

- 可以提交注册，但默认状态为 `pending`。
- 管理员审核为 `active` 后才允许登录。
- 可以查看分配给自己的预约。
- 可以更新自己的预约状态。
- 可以生成或更新自己负责预约的体检报告。
- 可以查看与预约、报告相关的客户档案。
- 不能审核医生，不能管理系统用户状态。

管理员 `admin`：

- 由种子数据初始化，不开放公开注册。
- 可以查看用户列表。
- 可以审核医生账号。
- 可以停用账号。
- 可以管理机构、套餐、体检项目、套餐项目组合和医生号源。

## 后端防线

- 后端必须校验 JWT、Redis session、用户状态和角色权限。
- 前端隐藏菜单不能作为权限控制依据。
- 用户越权访问他人预约或报告必须返回 `403`。
- 医生越权访问非本人预约或报告必须返回 `403`。
- 未登录访问受保护接口必须返回 `401`。
- 登录、注册、提交预约、生成报告等操作必须防重复提交。
- 后端启用同 IP 限流，避免恶意或误操作高频请求。
